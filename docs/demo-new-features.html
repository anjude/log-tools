紧凑显示

每个日志文件一行，节省空间

完整路径

显示完整文件路径和目录结构

📱 功能1：日志文件区域紧凑显示

每个日志文件现在只占用一行，大大节省了垂直空间，文件信息更加紧凑。

日志文件列表（紧凑显示）

🔍 功能2：日志文件列表搜索功能

在文件列表顶部添加了搜索输入框，支持按文件名和路径搜索，实时过滤结果。

搜索特性：
  • 实时搜索，输入时即时过滤
  • 支持文件名搜索
  • 支持完整路径搜索
  • 智能显示/隐藏目录标题
  • 不区分大小写
搜索示例：
输入 "app" → 找到所有包含 "app" 的文件
输入 "error" → 找到所有包含 "error" 的文件
输入 "logs" → 找到路径中包含 "logs" 的文件
清空搜索框 → 恢复显示所有文件

📁 功能3：完整文件路径显示

鼠标悬停在文件上可看到完整文件路径，文件按目录分组显示，便于管理。

显示信息：
  • 文件名：显示日志文件的基本名称
  • 修改时间：显示文件的最后修改时间
  • 文件大小：显示文件大小（自动格式化）
  • 完整路径：鼠标悬停显示完整文件路径
  • 所属目录：按目录分组显示文件
排序逻辑：
1. 首先按目录名称排序
2. 同一目录内按文件名排序
3. 提供更直观的文件组织结构

⚙️ 技术实现

后端改进：
type LogFile struct { Path string // 相对路径 Name string // 文件名 FullPath string // 完整文件路径 Directory string // 所属目录 Size int64 ModTime string }
前端改进：
  • 紧凑的单行显示
  • 目录标题分组
  • 文件图标和状态指示
  • 实时搜索过滤
  • 悬停效果和视觉层次

🚀 使用方法

启动服务：
# 编译并运行 go build -o log-tools.exe . ./log-tools.exe # 或直接运行 go run .
访问界面：
  1. 打开浏览器访问 http://localhost:6003
  2. 使用配置的用户名和密码登录
  3. 在左侧文件列表中使用搜索功能
  4. 点击文件查看日志内容
